III

바이트코드 가상 머신

자바 인터프리터인 jlox는 프로그래밍 언어의 많은 기초를 가르쳐 주었지만, 아직 배울 것이 많습니다. 먼저, Jlox에서 흥미로운 Lox 프로그램을 실행하면 매우 느리다는 것을 알게 될 것입니다. AST를 직접 걷는 방식은 실제 사용에는 충분하지만 범용 스크립트 언어에는 아쉬운 점이 많습니다.

또한, 우리는 암묵적으로 JVM 자체의 런타임 기능에 의존합니다. 우리는 다음과 같은 것들을 당연하게 여깁니다 instanceof 자바 작업에서 어떻게든. 그리고 우리는 메모리 관리에 대해 잠시도 걱정하지 않습니다. 왜냐하면 JVM의 쓰레기 수집가가 그것을 대신 처리해주기 때문입니다.

우리가 고차원적인 개념에 집중할 때, 그것들을 얼버무려도 괜찮았습니다. 하지만 이제 통역사를 찾는 방법을 알게 되었으니 이제는 C 표준 라이브러리를 사용하여 하위 계층까지 파고들어 처음부터 가상 머신을 구축할 때입니다. . .
