A1

부록 I

다음은 Lox의 완전한 문법입니다. 언어의 각 부분을 소개하는 장에는 문법 규칙이 포함되어 있지만, 이렇게 하면 모든 규칙이 한 곳에 모입니다.

A1 . 1구문 문법

구문 문법은 토큰의 선형 시퀀스를 중첩 구문 트리 구조로 구문 분석하는 데 사용됩니다. 전체 Lox 프로그램(또는 단일 REPL 항목)과 일치하는 첫 번째 규칙으로 시작합니다.

program        → declaration* EOF ;

A1 . 1 . 1선언문

프로그램은 새로운 식별자나 다른 문 유형을 묶는 일련의 선언문입니다.

declaration    → classDecl
               | funDecl
               | varDecl
               | statement ;

classDecl      → "class" IDENTIFIER ( "<" IDENTIFIER )?
                 "{" function* "}" ;
funDecl        → "fun" function ;
varDecl        → "var" IDENTIFIER ( "=" expression )? ";" ;

A1 . 1 . 2진술들

나머지 문장 규칙들은 부작용을 일으키지만, 바인딩을 도입하지는 않습니다.

statement      → exprStmt
               | forStmt
               | ifStmt
               | printStmt
               | returnStmt
               | whileStmt
               | block ;

exprStmt       → expression ";" ;
forStmt        → "for" "(" ( varDecl | exprStmt | ";" )
                           expression? ";"
                           expression? ")" statement ;
ifStmt         → "if" "(" expression ")" statement
                 ( "else" statement )? ;
printStmt      → "print" expression ";" ;
returnStmt     → "return" expression? ";" ;
whileStmt      → "while" "(" expression ")" statement ;
block          → "{" declaration* "}" ;

참고로 block 는 문 규칙이지만 함수체와 같은 다른 몇 가지 규칙에서는 비터미널로도 사용됩니다.

A1 . 1 . 3표현.

표현은 가치를 만들어냅니다. Lox에는 다양한 수준의 우선순위를 가진 여러 단항 연산자와 이진 연산자가 있습니다. 일부 언어 문법은 우선순위 관계를 직접 인코딩하지 않고 다른 언어로 지정합니다. 여기서는 각 우선순위 수준에 대해 별도의 규칙을 사용하여 명확하게 설명합니다.

expression     → assignment ;

assignment     → ( call "." )? IDENTIFIER "=" assignment
               | logic_or ;

logic_or       → logic_and ( "or" logic_and )* ;
logic_and      → equality ( "and" equality )* ;
equality       → comparison ( ( "!=" | "==" ) comparison )* ;
comparison     → term ( ( ">" | ">=" | "<" | "<=" ) term )* ;
term           → factor ( ( "-" | "+" ) factor )* ;
factor         → unary ( ( "/" | "*" ) unary )* ;

unary          → ( "!" | "-" ) unary | call ;
call           → primary ( "(" arguments? ")" | "." IDENTIFIER )* ;
primary        → "true" | "false" | "nil" | "this"
               | NUMBER | STRING | IDENTIFIER | "(" expression ")"
               | "super" "." IDENTIFIER ;

A1 . 1 . 4유틸리티 규칙

위의 규칙을 조금 더 깔끔하게 유지하기 위해 일부 문법은 재사용되는 몇 가지 도우미 규칙으로 나뉩니다.

function       → IDENTIFIER "(" parameters? ")" block ;
parameters     → IDENTIFIER ( "," IDENTIFIER )* ;
arguments      → expression ( "," expression )* ;

A1 . 2어휘 문법

어휘 문법은 스캐너가 문자를 토큰으로 그룹화하는 데 사용됩니다. 구문이 문맥에 구애받지 않는 경우 어휘 문법은 규칙적입니다—재귀적 규칙이 없다는 점에 유의하세요.

NUMBER         → DIGIT+ ( "." DIGIT+ )? ;
STRING         → "\"" <any char except "\"">* "\"" ;
IDENTIFIER     → ALPHA ( ALPHA | DIGIT )* ;
ALPHA          → "a" ... "z" | "A" ... "Z" | "_" ;
DIGIT          → "0" ... "9" ;
